IT Admin Guide

Hyper-V & Oracle Database (Mobile Ready)

🛠️ 1. การติดตั้ง Hyper-V

ตรวจสอบใน BIOS ว่าเปิด Virtualization (VT-x / AMD-V) แล้ว

  1. เปิด Server Manager > คลิก Add roles and features
  2. เลือก Role-based installation และติ๊กที่ช่อง Hyper-V
  3. Virtual Switches: เลือก Network Card (NIC) ที่จะให้ VM ออกเน็ต
  4. Migration: ติ๊ก "Allow this server to send and receive live migrations..."
  5. Default Stores: กำหนดที่เก็บ VHDX (แนะนำ Drive อื่นที่ไม่ใช่ C:)
  6. กด Install และ Restart เครื่อง 1 รอบ
💾 2. การ Backup VM

วิธีที่ 1: Windows Server Backup (ฟรีในตัว)

  1. ติดตั้ง Feature: Windows Server Backup
  2. เปิดโปรแกรม > เลือก Backup Schedule > Custom
  3. Add Items > เลือกที่ Hyper-V > เลือก VM
  4. กำหนดเวลาและ Destination (เช่น External HDD)

วิธีที่ 2: Veeam Community Edition แนะนำที่สุด!

  • จุดเด่น: เป็นโปรแกรมฟรีที่ทำ Item-level recovery (ดึงไฟล์ย่อยจาก VM ได้) และ Backup ได้ขณะที่ VM เปิดอยู่โดยไม่สะดุด
🔄 3. การทำ Replication (DR Site)

การทำสำเนา VM ไปอีกเครื่อง เผื่อเครื่องหลักพัง

Step 1: ตั้งค่า Replica Broker (ทั้ง 2 เครื่อง)

  1. Hyper-V Settings > Replication Configuration
  2. ติ๊ก Enable this computer as a Replica server
  3. เลือก Use Kerberos (HTTP) และ "Allow replication from any authenticated server"

Step 2: เริ่มทำ Replication

  1. คลิกขวาที่ VM เครื่องต้นทาง > Enable Replication
  2. ใส่ชื่อ Server ปลายทาง และเลือก VHDX
  3. ตั้งความถี่ (เช่น 5 นาที) > กด Finish
🔌 4. แก้ปัญหา VM มองไม่เห็น Network

หากเครื่อง VM ไม่ออกเน็ต หรือหลุดบ่อย ให้เช็ค 3 จุดหลัก:

  • VLAN ID: หากใช้ VLAN ต้องไปที่ Settings VM > Network Adapter > ติ๊ก Enable virtual LAN identification และใส่เลข VLAN
  • บั๊ก VMQ (สำคัญมาก): อาการเน็ตสุ่มหลุด วิธีแก้คือไปที่ Settings VM > Network Adapter > Hardware Acceleration > เอาติ๊กถูกที่ Enable virtual machine queue ออก
  • MAC Address Spoofing: หาก VM รัน Load Balance หรือ VPN ต้องเปิด Enable MAC address spoofing ใน Advanced Features
🛢️ 5. สเปค VM สำหรับ Oracle Database

กฎเหล็กสำหรับการรัน Oracle 19c บน Hyper-V:

  • CPU: 2-4 vCPU ขึ้นไป
  • RAM: ขั้นต่ำ 4-8 GB
    ห้ามใช้ Dynamic Memory ให้ตั้งเป็น Static RAM เพื่อให้ Oracle จองพื้นที่ Memory ได้เสถียร
  • Storage: แยก Drive OS และ Database
    ต้องใช้ VHDX แบบ Fixed Size ห้ามใช้ Dynamically expanding เพื่อรักษาความเร็ว I/O ป้องกันระบบช้า
🗄️ 6. Backup & Replicate Oracle (ฉบับฟรี)

การ Backup Oracle

  • RMAN แนะนำ: Tool ฟรีของ Oracle Backup ได้ขณะเปิดระบบ (Hot Backup)
  • Data Pump (expdp): สำหรับ Backup ดึงเฉพาะบาง Table/Schema
⚠️ ระวัง: อย่าใช้ Veeam Snapshot ตัว Database โดยตรง อาจทำให้ไฟล์ Corrupt ควรให้ RMAN แบ็คอัพลงโฟลเดอร์ก่อน แล้วค่อยใช้ Veeam ก๊อปปี้ไฟล์นั้นไป

การทำ DR Site (ทดแทน Data Guard)

  • Hyper-V Replica: ง่ายสุด แต่ตอนเปิดใช้งานฝั่ง DR อาจต้องสั่ง Recover Database นิดหน่อย
  • Log Shipping Script: เขียน Script โยนไฟล์ Archive Logs ไปอีกเครื่อง แล้วสั่ง Apply อัตโนมัติ (คนใช้ Standard Edition นิยมสุด)
💻 7. PowerShell Cheat Sheet 
# ดูสถานะ VM ทั้งหมด
Get-VM

# เช็คสถานะ Replication
Get-VMReplication

# Start VM ที่ปิดอยู่ทั้งหมด
Get-VM | where {$_.State -eq 'Off'} | Start-VM
💡 Best Practices ทั่วไป: ไม่ควรเก็บ Snapshot/Checkpoint ของ VM ทิ้งไว้นานเกิน 24-48 ชม. เพราะจะกินพื้นที่ดิสก์และทำให้ระบบหน่วงครับ
