IT Admin Guide

Hyper-V & Oracle Database (Enterprise Free Edition)

🛠️ 1. การติดตั้ง Hyper-V แบบละเอียด

ตรวจสอบใน BIOS ของ Server ว่าเปิด Virtualization (VT-x / AMD-V) เรียบร้อยแล้ว

  1. เปิด Server Manager บน Windows Server
  2. มุมขวาบนคลิก Manage > เลือก Add roles and features
  3. กด Next ไปเรื่อยๆ จนถึงหน้า Server Roles ให้ติ๊กถูกที่ช่อง Hyper-V
  4. หน้า Virtual Switches: เลือกว่าจะให้ VM ใช้การ์ดแลน (NIC) ใบไหนเพื่อออกเน็ต
  5. หน้า Migration: ติ๊กถูกที่ "Allow this server to send and receive live migrations..."
  6. หน้า Default Stores: สำคัญมาก! ให้เปลี่ยน Path ที่เก็บไฟล์ VHDX และ Configuration ไปไว้ที่ Drive อื่นที่ไม่ใช่ Drive C: (เช่น D:\Hyper-V)
  7. กด Install และ Restart เครื่อง 1 รอบ

💾 2. วิธีการ Backup VM

วิธีที่ 1: ใช้ Windows Server Backup (ฟรีในตัว)

  1. ติดตั้ง Feature Windows Server Backup ผ่าน Server Manager
  2. เปิดโปรแกรม > เลือก Backup Schedule...
  3. เลือก Custom > Add Items > เลือกที่โฟลเดอร์ Hyper-V และเลือก VM
  4. กำหนดเวลา (ควรเป็นกลางคืน) และเลือก Backup ลง External Disk หรือ Network Share

วิธีที่ 2: ใช้ Veeam Community Edition แนะนำที่สุด

  • ติดตั้ง Veeam Backup & Replication (ตัวฟรี)
  • เพิ่ม Hyper-V Server เข้าไปในระบบ
  • สร้าง Backup Job และเลือก VM ที่ต้องการ แนะนำให้เปิดฟีเจอร์ Application-aware processing เพื่อให้ Backup ทำงานร่วมกับ Database ภายใน VM ได้สมบูรณ์

🔄 3. การทำ Replication (DR Site ระดับ VM)

การโยนก้อน VM ไปสแตนด์บายที่ Server อีกเครื่องแบบ Real-time

Step 1: ฝั่งรับ (เครื่องสำรอง)

  1. เปิด Hyper-V Settings > Replication Configuration
  2. ติ๊ก Enable this computer as a Replica server
  3. เลือก Use Kerberos (HTTP) และใส่ IP ของเครื่องหลักในช่อง "Allow replication from the specified servers"

Step 2: ฝั่งส่ง (เครื่องหลัก)

  1. คลิกขวาที่ VM เครื่องหลัก > Enable Replication...
  2. ใส่ชื่อหรือ IP ของเครื่องสำรอง
  3. เลือก VHDX ที่ต้องการซิงค์
  4. ตั้งความถี่ (เช่น 5 นาที) แล้วกด Finish ระบบจะเริ่มโยนข้อมูลก้อนแรก (Initial Sync) ทันที

🔌 4. แก้ปัญหา VM มองไม่เห็น Network

1. ปิด VMQ (Virtual Machine Queue) ปัญหายอดฮิต

ถ้า VM เน็ตหลุดสุ่มๆ (โดยเฉพาะการ์ดแลน Broadcom) ให้แก้ดังนี้:

  1. คลิกขวาที่ VM > Settings...
  2. เมนู Network Adapter > Hardware Acceleration
  3. ด้านขวามือ เอาเครื่องหมายถูกออก จาก Enable virtual machine queue

2. การใส่ VLAN ID

  • ไปที่ Settings VM > Network Adapter > ติ๊ก Enable virtual LAN identification แล้วใส่เลข VLAN ให้ตรงกับสวิตช์ขององค์กร

3. MAC Address Spoofing

  • หาก VM รัน Docker/Kubernetes, Load Balance หรือ VPN ต้องกางเมนู Network Adapter ออกมาเลือก Advanced Features แล้วติ๊ก Enable MAC address spoofing

🛢️ 5. ตั้งค่า VM สำหรับ Oracle Database

การใช้ Hyper-V รัน Oracle (เช่น 19c) มีกฎเหล็กเพื่อป้องกันระบบล่มดังนี้:

1. การตั้งค่า Storage (Disk) บังคับทำ

  • Oracle ต้องการ I/O ดิสก์ที่นิ่งมาก ต้องสร้างไฟล์ VHDX เป็นแบบ "Fixed Size" เท่านั้น ห้ามใช้ Dynamically expanding (ดิสก์ขยายเอง) เด็ดขาด
  • ควรแยก Drive OS (C:) และ Drive ที่เก็บ Database Data (D:, E:) ออกจากกัน

2. การตั้งค่า Memory (RAM) บังคับทำ

  • ไปที่ Settings > Memory
  • เอาเครื่องหมายติ๊กถูกออก จากช่อง Enable Dynamic Memory
  • ใส่ Startup RAM คงที่ (เช่น 8192 MB) เพื่อให้ Oracle จองพื้นที่ SGA/PGA ได้อย่างเสถียร ไม่โดน Windows แย่ง RAM คืน

3. CPU (vCPU)

  • ขั้นต่ำ 2 Core (แนะนำ 4-8 Core ขึ้นไปสำหรับระดับ Production)

🛡️ 6. Backup Oracle (ฉบับฟรี)

⚠️ สำคัญมาก: อย่าใช้ Veeam/Hyper-V สั่ง Snapshot แบ็คอัพ VM ที่รัน Oracle อยู่โดยตรง (เสี่ยง Corrupt) ต้องสั่งทำ RMAN Backup ลงไฟล์ภายใน VM ก่อนเสมอ!

RMAN (Recovery Manager) แนะนำ

เครื่องมือระดับ Enterprise ที่มีมาให้ฟรีใน Oracle ทุกรุ่น ทำ Hot Backup (แบ็คอัพตอนเปิดใช้งาน) ได้

# เข้าสู่ระบบ RMAN
rman target /

# ตั้งค่าให้เซฟ Controlfile อัตโนมัติ
CONFIGURE CONTROLFILE AUTOBACKUP ON;

# สั่ง Backup Database และ Archive Log ทั้งหมด
BACKUP DATABASE PLUS ARCHIVELOG;

💡 แนะนำให้เขียน Script รันคำสั่งด้านบนทุกๆ คืน ตี 2 แล้วใช้ Veeam เก็บโฟลเดอร์ Backup นี้ไปอีกชั้นนึง

Data Pump (expdp / impdp)

ใช้สำหรับ Backup แบบ Logical คือเอาเฉพาะบาง Table, Schema หรือ Structure

# โยน Schema ออกเป็นไฟล์ .dmp
expdp system/password schemas=HR directory=BACKUP_DIR dumpfile=hr_backup.dmp

🔗 7. Replicate Oracle Database ฉบับฟรี

แทนที่จะเสียเงินซื้อ Oracle Enterprise เพื่อใช้ Data Guard เราสามารถทำ DR Site ฟรีได้ด้วย 3 วิธีหลักนี้:

วิธีที่ 1: Log Shipping (Manual Standby) ยอดนิยมสุด

เหมาะกับรุ่น Standard Edition อาศัยการโยนไฟล์ Log ข้ามเครื่อง (RPO ประมาณ 5-15 นาที)

  1. เครื่องหลัก (Primary): สั่งเปิด ALTER DATABASE ARCHIVELOG;
  2. เครื่องหลัก: เขียน Script (Batch) รันคำสั่ง robocopy ดึงไฟล์ Archive Log ใหม่ๆ ส่งไปโฟลเดอร์แชร์ของเครื่อง Standby ทุกๆ 5 นาที
  3. เครื่องสำรอง (Standby): ติดตั้ง Oracle และ Restore ข้อมูลก้อนแรกมาไว้ในสถานะ MOUNT
  4. เครื่องสำรอง: เขียน Script ลูปเพื่อสั่ง RECOVER AUTOMATIC STANDBY DATABASE; ให้มันนำไฟล์ Log ที่เพิ่งได้รับมายัดเข้า Database ทันที

วิธีที่ 2: Oracle SEHA (High Availability)

สำหรับ Oracle 19c Standard Edition 2 ขึ้นไป (ทดแทน RAC)

  1. สร้าง Shared Disk (เช่น ใช้ Shared VHDX ใน Hyper-V) ต่อเข้ากับ VM 2 เครื่อง
  2. ติดตั้ง Oracle Grid Infrastructure ทั้ง 2 เครื่อง และคอนฟิกแบบ Failover Cluster
  3. เก็บไฟล์ Database ไว้ตรงกลาง (Shared Disk)
  4. เมื่อเครื่อง A พัง ระบบจะสลับไป Start ฐานข้อมูลที่เครื่อง B อัตโนมัติ (IP เดิมไม่เปลี่ยน)

วิธีที่ 3: Logical Replication (CDC)

ใช้ Open Source Tools อย่าง SymmetricDS หรือ Debezium (Real-time)

  1. ติดตั้ง Agent ไว้ที่ทั้ง 2 เครื่อง
  2. ตั้งค่าให้โปรแกรมดึงเฉพาะตาราง (Tables) ที่ต้องการซิงค์
  3. ระบบจะจับการเปลี่ยนแปลง (Insert/Update) แล้วยิงไปอีกเครื่องแบบ Row-by-Row
  4. ข้อดี: เครื่องสำรองสามารถเปิดในสถานะ Read-Write ได้ และซิงค์ข้ามเวอร์ชัน Oracle ได้

💻 8. PowerShell Commands (Cheat Sheet)

# ดูสถานะ VM ทั้งหมดในเครื่อง
Get-VM

# ดูสถานะ Replication (DR Site)
Get-VMReplication

# เริ่มต้นเปิด VM ทั้งหมดที่โดนปิดอยู่
Get-VM | where {$_.State -eq 'Off'} | Start-VM

# ปิด VM อย่างปลอดภัย (Shutdown OS อย่างถูกต้อง)
Stop-VM -Name "Oracle_DB" -Force

# ตรวจสอบว่าไฟล์ VHDX สมบูรณ์หรือไม่
Test-VHD -Path "D:\Hyper-V\VM01.vhdx"
💡 ข้อแนะนำทิ้งท้าย: ไม่ควรเก็บ Snapshot (Checkpoint) ของ VM ทิ้งไว้นานเกิน 24 ชั่วโมง เพราะจะทำให้ไฟล์แตกแขนง กินพื้นที่ฮาร์ดดิสก์ และทำให้ I/O ของ Oracle ช้าลงอย่างหนักครับ
