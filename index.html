วิธีที่ 1: Log Shipping (Manual Standby) - ขั้นตอนละเอียด
🚨 ข้อควรระวังสูงสุด (Archive Log Management)
  • ORA-00257 (พื้นที่เต็ม): หาก Drive เก็บ Log ฝั่ง Primary เต็ม Database จะหยุดทำงานทันที (Hang)
  • Network Gap: หากเน็ตหลุด ไฟล์จะไม่ถูกส่ง และถ้าเราลบไฟล์ฝั่ง Primary ทิ้งก่อนส่ง ข้อมูลจะพังทันที
  • ลำดับไฟล์: ห้ามสลับลำดับ Sequence ของไฟล์ .arc เพราะ Standby จะไม่อ่านต่อ

ขั้นตอนการติดตั้ง (Step-by-Step)

  1. Step 1 [Primary] เปิดโหมด ARCHIVELOG และตั้ง Path แยก (ห้ามอยู่ Drive C:) 
    SQL> ALTER SYSTEM SET log_archive_dest_1='LOCATION=E:\Oracle\Archive' SCOPE=BOTH;
  2. Step 2 [Standby] สร้าง Share Folder และให้สิทธิ์ Write กับ User ที่ใช้รัน Script
  3. Step 3 [Primary] สร้าง Script ส่งไฟล์ (แนะนำใช้ Robocopy เพราะทนทานกว่า) 
    robocopy E:\Archive \\DR-Server\Archive /Z /MOT:5 /R:10 /W:30
  4. Step 4 [Standby] รันคำสั่ง Apply Log แบบอัตโนมัติ 
    SQL> RECOVER AUTOMATIC STANDBY DATABASE;

การตรวจสอบประจำวัน (Daily Checklist)

  • ตรวจสอบสถานะ Sequence ล่าสุดทั้ง 2 ฝั่ง (ต้องห่างกันไม่เกิน 1-2 ไฟล์) 
    SQL> SELECT MAX(SEQUENCE#) FROM V$ARCHIVED_LOG WHERE APPLIED='YES';
  • ตั้งค่า RMAN Housekeeping เพื่อลบไฟล์ที่ส่งแล้วเท่านั้น 
    RMAN> DELETE ARCHIVELOG ALL COMPLETED BEFORE 'SYSDATE-2';
วิธีที่ 2: Oracle SEHA (High Availability)

ขั้นตอนการติดตั้ง

  1. สร้าง Shared VHDX ใน Hyper-V และ Mount เข้า VM 2 เครื่องพร้อมกัน
  2. ติดตั้ง Grid Infrastructure (ASM) เพื่อทำ Shared Storage
  3. ติดตั้ง Oracle DB ลงบน ASM Disk
  4. ใช้คำสั่ง srvctl add database เพื่อให้ Clusterware รู้จักและย้ายคิวรันได้อัตโนมัติ

ข้อดี: ข้อมูลไม่หาย (Zero Data Loss) แต่อุปกรณ์ Shared Storage ต้องเสถียรมาก

วิธีที่ 3: Logical Replication (CDC - SymmetricDS)

ขั้นตอนการติดตั้ง

  1. เปิด Supplemental Logging เพื่อให้ Redo Log บันทึกรายละเอียดการแก้ Row
  2. ติดตั้ง SymmetricDS Java Agent ไว้ที่เครื่อง Master
  3. Config ไฟล์ engine.properties ระบุ DB URL และ User/Pass
  4. รันคำสั่ง symadmin create-sym-tables เพื่อให้ Tool สร้างตารางมอนิเตอร์ใน Oracle

ดีที่สุดสำหรับ: การดึงข้อมูลไปเข้าฐานข้อมูลยี่ห้ออื่น หรือทำระบบ Dashboard แยก

🏆 สรุป: วิธีไหนดีที่สุด?

1. Log Shipping: ดีสุดสำหรับ "งบประหยัด" แต่ Admin ต้องคอยดู Disk Space
2. SEHA: ดีสุดสำหรับ "ข้อมูลสำคัญห้ามหาย" แต่ต้องมี Storage กลาง
3. CDC: ดีสุดสำหรับ "การเชื่อมต่อระบบอื่น" หรือทำ Real-time Report
