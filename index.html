📈 8. ระบบ Monitoring & การแจ้งเตือน (Alerting)

เพื่อป้องกันปัญหาระบบล่มโดยไม่รู้ตัว (เช่น พื้นที่ Log เต็ม หรือ Service ดับ) การมีระบบ Monitoring จึงจำเป็นอย่างยิ่งสำหรับ IT Admin ครับ

ตาข่ายชั้นที่ 1: ใช้ Checkmk มอนิเตอร์แบบรวมศูนย์

Checkmk (Raw Edition) เป็นเครื่องมือ Open Source ที่ยอดเยี่ยมมาก สามารถตั้ง Checkmk Server บน VM Ubuntu 22.04 LTS ได้ฟรี และมอนิเตอร์อุปกรณ์ได้ครอบคลุม

  1. การมอนิเตอร์ Hyper-V Host: ติดตั้ง Checkmk Agent บน Windows Server แล้วนำไฟล์ hyperv_vms.ps1 ไปใส่ในโฟลเดอร์ plugins เพื่อให้ระบบแสดงสถานะ VM ทุกตัวว่าเปิดหรือปิดอยู่ พร้อมดูการกิน CPU/RAM ของแต่ละ VM ได้
  2. การมอนิเตอร์ Oracle Database: นำปลั๊กอิน mk_oracle ไปวางที่ฝั่ง Database Server กำหนดสิทธิ์ User ให้เรียบร้อย Checkmk จะสามารถมอนิเตอร์พื้นที่ Tablespace, สถานะ Archive Log, และการทำ Backup ของ RMAN ได้ลึกถึงแก่น
  3. Network & Firewall: สามารถใช้ SNMP มอนิเตอร์ Traffic ของ Switch หรือ Firewall (เช่น Zyxel USG) เพื่อดู Bandwidth ที่ใช้ในการ Replicate ข้อมูลได้ด้วย
ตาข่ายชั้นที่ 2: เขียน Script แจ้งเตือนเข้า LINE / Telegram

คุณสามารถนำ Script (PowerShell หรือ Python) ไปผูกกับ Task Scheduler เพื่อให้เช็คสถานะสำคัญ แล้วยิง Webhook แจ้งเตือนเข้ามือถือทีม IT ได้ทันที

ตัวอย่าง PowerShell: เช็คพื้นที่ Archive Log แล้วแจ้งเตือน LINE

$Token = "ใส่_LINE_NOTIFY_TOKEN_ที่นี่"
$Drive = Get-WmiObject Win32_LogicalDisk | Where-Object { $_.DeviceID -eq "E:" }
$FreeSpaceGB = [math]::Round($Drive.FreeSpace / 1GB, 2)
$TotalSpaceGB = [math]::Round($Drive.Size / 1GB, 2)
$PercentFree = [math]::Round(($FreeSpaceGB / $TotalSpaceGB) * 100, 2)

if ($PercentFree -lt 20) {
    $Message = "⚠️ [ALERT] พื้นที่ Archive Log (Drive E:) เหลือเพียง $PercentFree% ($FreeSpaceGB GB) กรุณาตรวจสอบด่วน!"
    $Params = @{
        Uri = "https://notify-api.line.me/api/notify"
        Method = "Post"
        Headers = @{ "Authorization" = "Bearer $Token" }
        Body = @{ "message" = $Message }
    }
    Invoke-RestMethod @Params
}

ไอเดียเพิ่มเติม: คุณสามารถปรับโค้ดนี้เพื่อประยุกต์ใช้กับ Telegram API โดยเปลี่ยน URL เป็น https://api.telegram.org/bot[TOKEN]/sendMessage เพื่อแยกห้องแจ้งเตือนสำหรับ Admin / Tech / Staff ได้อย่างเป็นระบบ
