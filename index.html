Hyper-V Ultimate Guide

คู่มือฉบับสมบูรณ์: การติดตั้ง, Backup, DR Site และ Oracle Database

🛠️ 1. การติดตั้ง Hyper-V (Installation)

ก่อนเริ่ม ตรวจสอบใน BIOS ว่าได้เปิด Virtualization Technology (VT-x / AMD-V) เรียบร้อยแล้ว

  1. เปิด Server Manager > คลิก Add roles and features
  2. เลือก Role-based or feature-based installation
  3. เลือก Server ของคุณ และติ๊กที่ช่อง Hyper-V
  4. Virtual Switches: เลือก Network Card (NIC) ที่ต้องการทำเป็น Bridge เพื่อให้ VM ออกเน็ตได้
  5. Migration: ติ๊กถูกที่ "Allow this server to send and receive live migrations..."
  6. Default Stores: กำหนดที่อยู่ไฟล์ VHDX และ Configuration (แนะนำให้แยกไป Drive อื่นที่ไม่ใช่ C:)
  7. กด Install และ Restart เครื่อง 1 รอบ

💾 2. การ Backup VM

วิธีที่ 1: ใช้ Windows Server Backup (ฟรีในตัว)

  1. ติดตั้ง Feature: Windows Server Backup ใน Server Manager
  2. เปิดโปรแกรม > เลือก Backup Schedule
  3. เลือก Custom > Add Items
  4. เลือกที่ Hyper-V > ติ๊กเลือก VM ที่ต้องการ
  5. กำหนดเวลาและ Destination (เช่น External HDD หรือ Network Share)

วิธีที่ 2: ใช้ Veeam Community Edition แนะนำที่สุด! ฟรี

  • ติดตั้ง Veeam > เพิ่ม Hyper-V Server > สร้าง Backup Job
  • จุดเด่น: ทำ Item-level recovery (ดึงไฟล์ย่อยออกมาได้) และ Backup ได้ขณะที่ VM เปิดทำงานอยู่โดยระบบไม่สะดุด

🔄 3. การทำ Replication (DR Site ระดับ VM)

ส่งสำเนา VM จาก Host ต้นทางไป Host ปลายทาง เพื่อสลับไปใช้เวลาเครื่องหลักมีปัญหา

Step 1: ตั้งค่า Replica Broker (ทำทั้งสองเครื่อง)

  1. ใน Hyper-V Manager > Hyper-V Settings
  2. ไปที่ Replication Configuration
  3. ติ๊ก Enable this computer as a Replica server
  4. เลือก Use Kerberos (HTTP) หรือ Certificate-based Authentication
  5. ในส่วน Authorization เลือก "Allow replication from any authenticated server"

Step 2: เริ่มทำ Replication บน VM

  1. คลิกขวาที่ VM เครื่องต้นทาง > เลือก Enable Replication
  2. ใส่ชื่อ Replica Server (เครื่องปลายทาง)
  3. เลือก VHDX ที่ต้องการส่งไป
  4. ตั้งค่า Replication Frequency (เช่น 5 นาที)
  5. เลือก "Send initial copy over the network" > กด Finish

🌐 4. Network & คำสั่ง PowerShell ที่ใช้บ่อย

ประเภทของ Virtual Switch

  • External: เชื่อมต่อเน็ตเวิร์กภายนอก (ใช้ IP วงเดียวกับ Host)
  • Internal: คุยกันเองระหว่าง VM และ Host (Host มองเห็น)
  • Private: คุยกันเองเฉพาะ VM (Host มองไม่เห็น)

Cheat Sheet PowerShell

# ตรวจสอบสถานะ VM ทั้งหมด
Get-VM

# ตรวจสอบสถานะ Replication
Get-VMReplication

# สั่ง Start VM ทั้งหมดที่ปิดอยู่
Get-VM | where {$_.State -eq 'Off'} | Start-VM

# ตรวจสอบความสมบูรณ์ของ VHDX
Test-VHD -Path "D:\Hyper-V\VM01.vhdx"

🔌 5. การแก้ปัญหา "VM มองไม่เห็น Network"

หากเครื่อง VM ไม่ได้รับ IP หรือออกเน็ตไม่ได้ ให้เช็ค 4 จุดนี้:

  • Virtual Switch Type: ตรวจสอบว่าเลือกใช้ External Switch และผูกกับการ์ดแลน (NIC) ถูกใบ
  • VLAN ID: หากระบบเน็ตเวิร์กขององค์กรมีการทำ VLAN ต้องเข้าไปที่ Settings ของ VM > Network Adapter > ติ๊ก Enable virtual LAN identification และใส่เลข VLAN ให้ถูกต้อง
  • VMQ (Virtual Machine Queue) Bug: (เจอบ่อยมากใน Server บางรุ่น) อาการคือเน็ตหลุดสุ่มๆ วิธีแก้: ไปที่ Settings VM > Network Adapter > Hardware Acceleration > ปิด/เอาติ๊กถูกที่ Enable virtual machine queue ออก
  • MAC Address Spoofing: หาก VM นั้นทำหน้าที่เป็น Load Balancer หรือ VPN ต้องเข้าไปเปิด Enable MAC address spoofing ใน Advanced Features

🛢️ 6. แนวทางการตั้งค่า VM สำหรับ Oracle Database

การนำ Hyper-V มารัน Database ระดับ Enterprise (เช่น Oracle 19c) มีกฎเหล็กเพื่อป้องกันระบบล่มดังนี้:

  • CPU: ขั้นต่ำ 2 vCPU (แนะนำ 4 vCPU ขึ้นไปสำหรับ Production)
  • RAM: ขั้นต่ำ 4 - 8 GB
    ⚠️ สำคัญ ห้ามใช้ Dynamic Memory ให้ตั้งเป็น RAM แบบ Fixed (ตายตัว) เพื่อให้ Oracle จองพื้นที่ SGA/PGA ได้เสถียร
  • Storage (VHDX): ควรแยก Drive ระหว่าง OS และ Database Data
    ⚠️ สำคัญ ต้องสร้างไฟล์ VHDX เป็นแบบ "Fixed Size" เท่านั้น ห้ามใช้ Dynamically expanding เพื่อรักษาความเร็ว I/O ของดิสก์ให้สูงสุด

🗄️ 7. การ Backup & Replicate Oracle Database (ฉบับฟรี)

สำหรับองค์กรที่ไม่ได้ใช้ตัว Enterprise Edition หรือต้องการโซลูชันแบบไม่เสียค่า License เพิ่มเติม แนะนำแนวทางดังนี้:

🛡️ วิธีการ Backup Oracle

  • 1. RMAN (Recovery Manager) แนะนำที่สุด: เป็นเครื่องมือฟรีของ Oracle เอง รองรับการทำ Full, Incremental และ Hot Backup (Backup ได้โดยไม่ต้องปิดระบบ)
  • 2. Oracle Data Pump (expdp/impdp): เหมาะสำหรับการ Backup แบบ Logical เช่น ต้องการดึงเฉพาะบาง Schema หรือแค่บาง Table ข้ามไปอีกเครื่อง
  • 3. Cold Backup (OS Level): สั่งปิด Database (Shutdown) แล้วทำการ Copy ไฟล์ .dbf, Control files และ Redo logs ไปเก็บตรงๆ (เหมาะกับระบบที่ยอมให้มี Downtime นอกเวลาได้)
⚠️ ข้อควรระวัง: Veeam/Hyper-V Backup กับ Oracle
การสั่ง Snapshot หรือ Backup ระดับ VM อาจทำให้ไฟล์ Oracle Database "Corrupt" ได้ หากไม่ได้ติดตั้ง/คอนฟิก Oracle VSS Writer ให้ถูกต้อง วิธีที่ปลอดภัยที่สุดคือ ให้ RMAN ทำการ Backup ลงโฟลเดอร์ภายใน VM ก่อน แล้วค่อยใช้ Veeam/Hyper-V ดึงไฟล์ RMAN นั้นไปเก็บอีกที

🔄 วิธีการ Replicate Oracle ทำ DR Site (ทดแทน Data Guard)

เนื่องจากฟีเจอร์ Oracle Data Guard ต้องเสียเงินซื้อแบบ Enterprise เราจึงมีทางเลือกฟรีดังนี้:

  1. ใช้ Hyper-V Replica (OS-Level DR) ทำง่ายสุด:
    • ใช้ฟีเจอร์ Replication ของ Hyper-V ที่ทำไว้ในหัวข้อที่ 3
    • ข้อจำกัด: มันคือการทำ Crash-Consistent Backup (เสมือนดึงปลั๊กเครื่องหลักไปเปิดอีกเครื่อง) ตอนเปิดฝั่ง DR Site อาจต้องเข้า SQL*Plus ไปรันคำสั่ง RECOVER DATABASE เล็กน้อยเพื่อให้ข้อมูลสมบูรณ์
  2. Manual Standby Database (Log Shipping Script):
    • เขียน Script (Batch/Shell) เพื่อโยนไฟล์ Archive Logs จากเครื่องหลักไปยังเครื่องสำรองผ่าน Network Share (SMB/FTP)
    • ตั้งให้เครื่องสำรองรัน Script อัตโนมัติเพื่อนำ Log นั้นมา Apply เข้า Database (เป็นการทำ Near Real-time DR ที่คนใช้ Standard Edition นิยมที่สุด)
  3. ใช้ Open Source CDC (Change Data Capture):
    • ใช้ Tool ฟรีอย่าง Debezium หรือ SymmetricDS เพื่อจับการเปลี่ยนแปลงระดับ Row แล้วยิงข้อมูลข้ามไปยังอีก Database หนึ่งแบบ Real-time (ตั้งค่ายากกว่า แต่ยืดหยุ่นสูงและข้ามเวอร์ชัน Database ได้)
💡 ข้อควรระวัง (Best Practices) ทั่วไปของ Hyper-V
  • Memory: Server สำคัญควรใช้ Startup RAM แบบคงที่ (Fixed) ไม่ใช้ Dynamic
  • Checkpoints: ไม่ควรเก็บ Snapshot ไว้นานเกิน 24-48 ชม. เพราะจะกินพื้นที่และทำให้เครื่องช้า
  • Integration Services: อัปเดต Integration Services ใน VM เสมอ เพื่อให้ระบบ Backup ทำงานสมบูรณ์
