คู่มือแบบ Step-by-Step: Hyper-V & Oracle Database

🛠️ 1. การติดตั้ง Hyper-V แบบละเอียด

ตรวจสอบใน BIOS ของ Server ว่าเปิด Virtualization (VT-x / AMD-V) เรียบร้อยแล้ว

  1. เปิด Server Manager บน Windows Server
  2. มุมขวาบนคลิก Manage > เลือก Add roles and features
  3. กด Next ไปเรื่อยๆ จนถึงหน้า Server Roles ให้ติ๊กถูกที่ช่อง Hyper-V > กด Add Features
  4. หน้า Virtual Switches: เลือกว่าจะให้ VM ใช้การ์ดแลน (NIC) ใบไหนเพื่อออกเน็ต (หาก Server มีหลายพอร์ต ให้เลือกพอร์ตที่ต่อสายแลน External ไว้)
  5. หน้า Migration: ติ๊กถูกที่ "Allow this server to send and receive live migrations..." (หากมีแผนทำ Cluster ในอนาคต)
  6. หน้า Default Stores: สำคัญมาก! ให้เปลี่ยน Path ที่เก็บไฟล์ VHDX และ Configuration ไปไว้ที่ Drive อื่นที่ไม่ใช่ Drive C: (เช่น D:\Hyper-V)
  7. หน้า Confirmation ติ๊กเลือก Restart destination server automatically > กด Install (เครื่องจะรีสตาร์ท 1-2 ครั้ง)
💾 2. วิธีการ Backup VM
วิธีที่ 1: ใช้ Windows Server Backup (ฟรีในตัว)
  1. เปิด Server Manager > Add roles and features > หน้า Features ติ๊กเลือก Windows Server Backup > Install
  2. ไปที่ปุ่ม Start ค้นหาและเปิดโปรแกรม Windows Server Backup
  3. แถบ Action ด้านขวามือ คลิก Backup Schedule...
  4. หน้า Select Backup Configuration เลือก Custom
  5. หน้า Select Items for Backup คลิก Add Items > กางโฟลเดอร์ Hyper-V > ติ๊กเลือก VM ที่ต้องการ Backup
  6. กำหนดเวลา Backup (Specify Backup Time) แนะนำเป็นช่วงดึกที่ไม่มีการใช้งานระบบ
  7. เลือก Destination ให้เซฟลง Back up to a hard disk that is dedicated for backups (แนะนำสุด) หรือ Network Share
  8. ตรวจสอบความถูกต้องและกด Finish
วิธีที่ 2: ใช้ Veeam Community Edition (ฟรี / แนะนำ)
  1. ติดตั้ง Veeam Backup & Replication Community Edition ให้เรียบร้อย
  2. เปิด Veeam Console ไปที่เมนู Backup Infrastructure ด้านล่างซ้าย
  3. คลิกขวาที่ Managed Servers > Add Server > เลือก Microsoft Hyper-V
  4. ใส่ IP หรือ ชื่อของ Server เครื่องหลัก และระบุ Username/Password ของ Admin > กด Next จน Finish
  5. ไปที่เมนู Home ด้านล่างซ้าย > คลิก Backup Job (บน Ribbon) > เลือก Virtual machine
  6. ตั้งชื่อ Job > หน้า Virtual Machines กด Add... แล้วเลือก VM ที่ต้องการ
  7. หน้า Storage เลือก Backup Repository ที่เตรียมไว้
  8. หน้า Schedule ตั้งเวลาให้ทำงานทุกวัน (Daily) > กด Apply และ Finish
🔄 3. การทำ Replication (DR Site)

เงื่อนไข: ต้องมี Windows Server ที่ลง Hyper-V ไว้ 2 เครื่อง (เครื่องหลัก และ เครื่องสำรอง)

Step 1: ตั้งค่ารับข้อมูล (ทำที่เครื่องสำรอง DR)
  1. เปิด Hyper-V Manager บนเครื่องสำรอง > คลิกเมนู Hyper-V Settings... ด้านขวามือ
  2. ไปที่เมนู Replication Configuration
  3. ติ๊กถูกที่ช่อง Enable this computer as a Replica server
  4. หัวข้อ Authentication เลือกระบบความปลอดภัย (แนะนำ Use Kerberos (HTTP) Port 80 ถ้าระบบไม่ได้ต่อเน็ตเวิร์กภายนอก)
  5. หัวข้อ Authorization เลือก Allow replication from the specified servers > กด Add และใส่ IP ของเครื่องหลัก
  6. กด OK > *อย่าลืมไปตั้งค่า Windows Firewall ให้เปิด Port ให้ Hyper-V Replica HTTP Listener (TCP-In) ด้วย
Step 2: เริ่มส่งข้อมูล (ทำที่เครื่องหลัก)
  1. เปิด Hyper-V Manager บนเครื่องหลัก > คลิกขวาที่ VM ที่ต้องการ > เลือก Enable Replication...
  2. หน้า Replica Server ให้พิมพ์ IP หรือชื่อของเครื่องสำรอง DR
  3. หน้า Choose Replication VHDs ให้เลือกเฉพาะ Drive ที่ต้องการ (บาง Drive เช่น Temp ไม่จำเป็นต้องส่งไป)
  4. หน้า Configure Replication Frequency เลือกว่าจะส่งข้อมูลทุกๆ 30 วิ, 5 นาที หรือ 15 นาที
  5. หน้า Initial Replication เลือกว่าจะส่งข้อมูลก้อนแรกผ่าน Network ทันทีเลยหรือไม่ > กด Finish ระบบจะเริ่มซิงค์ข้อมูลทันที
🔌 4. แก้ปัญหา VM มองไม่เห็น Network
วิธีแก้ปัญหา VM หลุดบ่อย (ปิด VMQ)

ปัญหานี้พบบ่อยมากใน Server Enterprise บางรุ่นที่มีฟีเจอร์ Virtual Machine Queue ทำงานผิดปกติ

  1. ปิด VM (Shut Down) หรือทำตอนที่ระบบไม่มีคนใช้งาน
  2. เปิด Hyper-V Manager > คลิกขวาที่ VM ที่มีปัญหา > เลือก Settings...
  3. กางเมนู Network Adapter ออกมา > คลิกที่ Hardware Acceleration
  4. ด้านขวามือ ให้ เอาเครื่องหมายติ๊กถูกออก จากช่อง Enable virtual machine queue
  5. กด OK และเปิด VM ทดสอบดูอีกครั้ง
วิธีตั้งค่า VLAN ID ให้กับ VM
  1. เปิด Hyper-V Manager > คลิกขวาที่ VM > เลือก Settings...
  2. คลิกที่เมนู Network Adapter ด้านซ้าย
  3. ด้านขวามือในหมวด VLAN ID ให้ติ๊กถูกช่อง Enable virtual LAN identification
  4. พิมพ์ตัวเลข VLAN ID ขององค์กร (เช่น VLAN ของระบบกล้อง, ระบบเสียง, หรือแผนก) ในช่อง VLAN identifier
  5. กด OK
🛢️ 5. ตั้งค่า VM สำหรับระบบ ฐานข้อมูล (Oracle)
Step 1: การตั้งค่า Fixed VHDX (ห้ามใช้ Dynamic)

Oracle Database ต้องการ I/O ดิสก์ที่นิ่งมาก การใช้ดิสก์แบบขยายเอง (Dynamic) จะทำให้ระบบล่มหรือหน่วง

  1. ตอนสร้างดิสก์ใหม่ หรือใน Hyper-V Manager เลือกแถบขวามือ New > Hard Disk...
  2. เลือก Format เป็น VHDX
  3. หน้า Disk Type บังคับเลือก "Fixed size" เท่านั้น
  4. ระบุชื่อและตำแหน่ง (ควรแยกไปอยู่ Drive SSD/NVMe)
  5. ระบุขนาดที่ต้องการล่วงหน้า (เช่น 100GB) สร้างเสร็จจะได้ไฟล์ขนาด 100GB ทันที
Step 2: การตั้งค่า Static RAM
  1. คลิกขวาที่ VM > เลือก Settings...
  2. เลือกเมนู Memory ด้านซ้าย
  3. ช่อง Startup RAM ให้ใส่จำนวน RAM ที่ต้องการจัดสรรให้ DB (ขั้นต่ำ 4096 MB / แนะนำ 8192 MB ขึ้นไป)
  4. เอาเครื่องหมายติ๊กถูกออก จากช่อง Enable Dynamic Memory
  5. กด OK เพื่อบังคับให้ VM จอง RAM จำนวนนี้ไว้ตลอดเวลา
🗄️ 6. Backup & DR Site สำหรับ Oracle (ฉบับฟรี)
⚠️ สำคัญ: ห้ามใช้ Veeam/Windows Backup สั่ง Snapshot ถ่าย Backup VM ที่รัน Oracle อยู่โดยตรง (เสี่ยง Corrupt) ต้องสั่งทำ RMAN ลงเครื่องก่อนเสมอ!
วิธีที่ 1: การ Backup ด้วย RMAN (เครื่องมือฟรีของ Oracle)
  1. เปิด Command Prompt / Terminal ใน OS ที่ลง Oracle
  2. พิมพ์คำสั่งเข้า RMAN: rman target /
  3. ตั้งค่าให้ Backup Control File อัตโนมัติ: CONFIGURE CONTROLFILE AUTOBACKUP ON;
  4. สั่ง Backup ฐานข้อมูลทั้งหมดพร้อม Archive Log: BACKUP DATABASE PLUS ARCHIVELOG;
  5. ทริค: เขียนคำสั่งเหล่านี้เซฟเป็นไฟล์ `.bat` หรือ `.sh` แล้วใช้ Task Scheduler/Cronjob สั่งรันทุกคืน (เช่น ตี 2) แล้วค่อยใช้ Veeam มาเก็บโฟลเดอร์ Backup นี้ไปอีกที
วิธีที่ 2: ทำ DR Site สำรองข้อมูลข้ามเครื่อง (Log Shipping)

ใช้ทดแทน Oracle Data Guard ที่ต้องเสียเงิน โดยการโยนไฟล์ Log ข้ามเครื่อง (ทำ Near Real-time)

  1. เตรียม Server อีกเครื่อง (Standby) ติดตั้ง Oracle OS/Version ให้เหมือนเครื่องหลัก (Primary) เป๊ะๆ
  2. เปิด ARCHIVELOG Mode บนเครื่องหลัก
  3. ตั้งค่า Network Share Folder ให้เครื่องหลักมองเห็นเครื่อง Standby
  4. เขียน Script (Batch/Python) คอยจับไฟล์ .arc (Archive log) ใหม่ๆ ในเครื่องหลัก ก๊อปปี้โยนไปไว้ใน Share Folder ของเครื่อง Standby ทุกๆ X นาที
  5. ที่เครื่อง Standby รัน Script สั่ง RECOVER STANDBY DATABASE; เพื่อเอา Log ที่ส่งมา เข้าไปอัปเดตข้อมูลให้ตรงกับเครื่องหลัก
